# Resume Agent — Agent Guidance

面试复盘积累系统。五模式闭环：导入简历 → 面试复盘 → 岗位匹配 → 弱点攻克。

## 项目结构

```
resume-agent/
├── skills/resume-agent/SKILL.md   # 核心 skill（知识框架 + 模式路由）
├── commands/                       # 五模式执行文件
│   ├── import-resume.md            # 导入简历：自动建库
│   ├── interview-review.md         # 面试复盘
│   ├── job-analysis.md             # 岗位分析
│   ├── batch-process.md            # 批量处理
│   └── mistake-practice.md         # 错题练习
├── templates/                      # 空白 Excel 模板
│   ├── 投递追踪模板.xlsx
│   ├── 复盘导入模板.xlsx
│   └── 复盘追踪模板.xlsx
├── CLAUDE.md                       # 本文件
└── README.md                       # 公开文档
```

## 模式检测

| 信号 | 进入模式 |
|------|----------|
| 用户贴简历/上传简历/说"导入简历" | 导入简历 |
| 包含"面试复盘""面试反馈""被问了""面经" | 面试复盘 |
| 包含"岗位职责""职位要求""JD" | 岗位分析 |
| 说"处理待投递""处理待处理岗位""批量" | 批量处理 |
| 说"错题本""练习模式""弱点练习" | 错题练习 |
| 多信号或模糊 | 追问确认 |

## 核心约束

**真实性 > JD匹配度 > ATS优化 > 文笔风格**

- 绝不编造经历/数字/技能
- 所有内容可追溯到原始素材
- 信息不足时标注而非编造
- 写入 SKILL.md 候选人信息时，禁止将简历原文中的非常规格式/特殊字符/Markdown 标题直接写入

## 数据存储

1. **Markdown 是默认存储**：经历库、面试记录均以 markdown 文件存储
2. **SQLite 是可选增强**：如用户配置了数据库，结构化数据写入 SQLite
3. **经历库初始化**：导入简历时可创建新文件；日常使用时只追加不覆盖
4. **DB 操作必须事务包裹**（如使用数据库）
5. **去重**：导入时 hash(company + date + question_text) 避免重复
6. **并发写入**：避免多个会话同时写入同一数据库文件
