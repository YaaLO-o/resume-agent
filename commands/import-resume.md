# 导入简历 — 一键建库

用户贴简历文本（或上传简历文件）时触发。自动解析为结构化经历库。

## 触发

- 用户贴简历内容
- 用户说"导入简历""上传简历""这是我的简历"
- 用户上传简历 PDF/DOCX 文件

## Step 1：解析简历

从简历中提取：

1. **个人信息**：姓名、学校、专业、届别、联系方式
2. **教育背景**：学校、专业、GIF/排名、主修课程
3. **实习经历**（每段）：公司、岗位、时间、职责描述、量化成果
4. **项目经历**（每个）：项目名、角色、技术栈、成果数据
5. **竞赛/获奖**：名称、级别、时间、成果
6. **校园经历**：组织/社团、职务、成果
7. **技能清单**：工具、语言、框架、证书

## Step 2：生成候选人信息

将解析出的个人信息写入 `skills/resume-agent/SKILL.md` 的「候选人信息」部分：

```markdown
## 候选人信息

- {姓名}，{学校} {专业} {届别}
- 求职方向：{从经历推断的求职方向}
- 核心经历：{经历1概述} + {经历2概述} + {经历3概述}
```

## Step 3：生成经历库文件

将每段经历转化为 STAR 格式的原子块，写入对应的 markdown 文件：

### `database/internships.md`

```markdown
## {公司名} — {岗位名} ({时间段})

### STAR 原子块

**Situation**: {背景}
**Task**: {任务/目标}
**Action**: {具体做了什么，用强动词开头}
**Result**: {量化结果，如"DAU提升30%"}
```

### `database/projects.md`

```markdown
## {项目名} ({时间段})

**角色**: {角色}
**技术栈**: {技术}
**背景**: {为什么做}
**做了什么**: {具体行动}
**成果**: {量化结果}
```

### `database/competitions.md`

```markdown
## {竞赛名称} ({时间})

**级别**: {国家级/省级/校级}
**成果**: {获奖等级}
**角色**: {你的角色}
**核心工作**: {做了什么}
```

### `database/leadership.md`

```markdown
## {组织/社团名} — {职务} ({时间段})

**背景**: {组织规模/职责范围}
**做了什么**: {具体行动}
**成果**: {量化结果}
```

### `database/education.md`

```markdown
## {学校} — {专业} ({入学年}-{毕业年})

**学历**: {本科/硕士}
**GPA/排名**: {如有}
**核心课程**: {与求职方向相关的课程}
```

### `database/skills.md`

按求职方向分组：

```markdown
## {求职方向1}
- {技能1}
- {技能2}

## {求职方向2}
- {技能1}
- {技能2}
```

## Step 4：生成 master-resume.md

汇总所有经历为一份总览文件 `database/master-resume.md`：

```markdown
# {姓名} — 简历总览

## 个人信息
...

## 求职方向
...

## 核心经历
1. {经历1}（优先级高）
2. {经历2}
3. {经历3}

## 技能矩阵
| 方向 | 技能 |
|------|------|
```

## Step 5：输出确认

在对话中展示：

1. 解析出的个人信息摘要
2. 识别到的所有经历（实习/项目/竞赛/校园）
3. 推断的求职方向
4. 生成的文件清单

> 简历已导入！共识别到 X 段实习、X 个项目、X 个竞赛。已生成经历库文件。
> 
> 你可以：
> - 贴 JD 开始岗位分析
> - 贴面经开始面试复盘
> - 补充项目细节（告诉我某个项目的具体数字）

## 注意事项

- 如果简历中某些信息不完整（如缺量化结果），标注为 `{待补充}` 而非编造
- 如果用户同时提供了项目补充材料，合并到对应经历中
- 经历库文件追加写入，不覆盖已有内容
