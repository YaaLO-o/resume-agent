# Resume Agent — 面试复盘积累系统

专注「面试复盘 → 结构化积累 → 弱点攻克」闭环。所有面试数据存入 SQLite，经历库（markdown）保持人类可读。

## 快速上手

```
第 1 步：贴简历 → 自动建库
第 2 步：贴 JD → 自动匹配经历 + 生成备战方案
第 3 步：贴面经 → 自动复盘 + 积累弱点
第 4 步：错题本 → 攻克弱点
```

## 项目路径

```
~/.claude/resume-agent/           # 用户数据目录（需自行创建）
├── CLAUDE.md                     # 项目说明书
├── database/                     # 经历库 (markdown) + SQLite
├── scripts/                      # Python 数据层（可选增强）
├── system/                       # 规则引擎
├── rules/                        # 匹配规则
├── 投递追踪.xlsx                  # 投递追踪主表
└── 复盘追踪.xlsx                  # 岗位复盘追踪表
```

> **注意**：本 skill 为纯 Markdown 包，不含 Python 脚本和数据文件。用户需自行搭建数据目录结构。脚本为可选增强，核心功能由 Claude 直接完成。

## 核心约束

**真实性 > JD匹配度 > ATS优化 > 文笔风格**

- 绝不编造任何经历、项目、实习、奖项、数字
- 绝不添加候选人不具备的技能
- 所有内容必须可追溯到 database/ 中的原始素材
- 信息不足时标注而非编造

## 权限边界

**允许**：重排经历顺序、改写措辞、压缩展开描述、弱动词换强动词、同一经历不同岗位强调不同侧面

**禁止**：编造经历/数字/技能、夸大职责范围、为凑关键词虚构细节

- 核心经历：{经历1概述} + {经历2概述} + {经历3概述}

## 数据存储规则

1. **SQLite 是主存储**（如配置）：结构化面试数据写入数据库
2. **Markdown 是镜像**：`interview-log.md` 保持人类可读格式
3. **经历库只读**：经历文件只在发现新事实时追加，不覆盖原有内容
4. **DB 操作必须事务包裹**（如使用数据库）
5. **Excel 操作用 openpyxl**（如配置）
6. **去重**：导入时 hash(company + date + question_text) 避免重复

## 反馈回路

```
面试复盘(模式一) → SQLite + interview-log.md
  → 岗位分析(模式二)检索历史 → 面试备战警告
  → 面试 → 复盘 → 循环
  → 错题练习(模式四)攻克弱点
```

每次复盘都在喂养下一次面试准备。
