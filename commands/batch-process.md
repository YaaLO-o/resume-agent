# 批量处理

用户说"处理待投递""处理待处理岗位""批量导入"或上传 Excel/docx 文件时触发。

## 准备工作

确认投递追踪表存在：
- 检查工作目录是否有 `投递追踪.xlsx`
- 若无，提示用户：「未找到投递追踪表。请复制 `templates/投递追踪模板.xlsx` 到当前工作目录，重命名为 `投递追踪.xlsx`（去掉"模板"二字），然后填入你的投递信息。」

## Step 1：判断输入类型

- **用户说"处理待处理岗位"/"处理待投递"** → 读取 `投递追踪.xlsx` 的「投递追踪」sheet，找到需要处理的行：
  1. **状态=待处理** → 必须处理
  2. **匹配方向为空 AND 有JD内容** → 必须处理
  3. **岗位类型为空 AND 有JD内容** → 必须处理
  4. **公司字段为空但 JD 中有公司信息** → 从 JD 原文提取公司名
- **上传 .docx 文件** → 提取文本 → 拆分 JD → 填入投递追踪 → 执行 Step 2
- **上传 Excel 文件（面试复盘数据）** → 参照 `templates/复盘导入模板.xlsx` 的列格式解析 → 逐条处理

## Step 2：处理待处理岗位 — 运行全流程岗位分析

**批量限制**：单次最多处理 10 条岗位。若待处理超过 10 条，先处理前 10 条，提示用户处理完后再次执行。

对每条待处理行，执行**岗位分析完整流水线**（`commands/job-analysis.md` Step 1-12，含输出展示）：

1. 解析 JD → 提取公司、岗位、关键词
2. 四维度匹配经历 → Top 3-5
3. 面试备战警告
4. 能力缺口与补齐方案 → 写入「补齐方案」sheet
5. 项目改造与复盘 → 写入「项目复盘」sheet
6. 产品思维训练 → 写入「思维训练」sheet
7. 面试前准备清单 → 写入「面试准备」sheet
8. 简历 Bullet → 写入「简历Bullet」sheet
9. 回填投递追踪表

**错误隔离**：每条岗位独立处理。若某条岗位中途失败：
- 记录失败原因
- 跳过该条，继续处理下一条
- 在汇总报告中标注失败项

## Step 3：汇总报告

```
批量处理完成：成功 X/Y

成功：
- 公司A | 岗位A | 类型 → 已写入
- 公司B | 岗位B | 类型 → 已写入

失败（如有）：
- 公司C | 岗位C | 失败原因：xxx

投递追踪表已回填。所有 sheet 已同步更新。

提示：简历 Bullet 已在上方逐条展示。如需查看完整分析，请对单个岗位使用岗位分析命令。
```
